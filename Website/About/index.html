About PhoediX and a Breif History
PhoediX (pronounced like PHOto EDitor X) is a free, open source, non destructive image editing application, available for Windows, macOS, and Linux Ubuntu. Hi, my name is Jacob Chesley and this is a personal project of mine. I like to work on this for fun in my spare time.

In 2013 during College at the University of Cincinnati, I took a course in Photography to fulfil a fine arts credit. During this course, I began working on an image editing program that I used to edit my photos for the class. After working on this project for fun on and off for 3 years, I rebuilt the application from the ground up in 2016 and named it PhoediX. PhoediX is the product of 5 years expirementation in image editing software creation.

PhoediX is written entireley in C++ for performance reasons. With each edit and many features multithreaded on all platforms, and careful memory management, PhoediX is created with performance as a top priorirty. Be sure to check out the source code repository at www.github.com/jacobchesley/PhoediX.
Libraries Used in PhoediX
wxWidgets
wxWidgets is the GUI library PhoediX uses, to maintain a consistent look and feel accross all platforms. wxWidgets is also used for image loading and saving (except 16 bit TIFF), and threading. Because of wxWidgets, PhoediX can be built on all major platforms with only a handfull of platform specific function calls, in one single codebase. For more information, visit www.wxwidgets.org
LibRaw
LibRaw is used in PhoediX to read, and render RAW image formats from over 1000 camera makes and models. When a RAW image is loaded, PhoediX passes the file to LibRaw internally to handle reading and rendering the output image. If the edit at the top of the Edit Pipeline is Raw Processor, the result image from the Raw Processor is a rendering from LibRaw. For more information, visit www.libraw.org
LibTiff
LibTiff is used to read and write 16 bit TIFF files. Though wxWidgets does have the capability to read 16 bit TIFF files, they are intenrally truncated to 8 bit images (all wxWidget images are stored in 8 bit format). If a 16 bit TIFF file is detected during load, PhoediX uses LibTiff to read the image and will store it in memory in 8 or 16 bit format (depending on the bit depth setting). Anytime a 16 bit TIFF is to be exported, LibTiff is used to write the file. For more information, visit www.simplesystems.org/libtiff
